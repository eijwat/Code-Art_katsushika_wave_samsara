# 葛飾の波 — 生々流転

### Katsushika Wave: Samsara

---

前作「創発」で波は生まれた。

しかし、生まれただけでは足りなかった。波には喜びも悲しみもなかった。ただ美しいだけだった。美しいだけのものは、まだ芸術ではない。

だから、この海に **感情** を与えた。

まずは、みてみよう。[GitHub Pages](https://eijwat.github.io/Code-Art_katsushika_wave_samsara/)

## 三つの層

### 一、死の残像（Ghost）

粒子は永遠には生きられない。`maxLife` を迎えた粒子は消える——しかし、ただ消えるのではない。その場に淡い光の残像を残し、ゆっくりと上へ昇っていく。最期の瞬間、粒子は明滅する。まるで最後の呼吸のように。

これが悲しみの層。

すべての美しいものは消える。消えるからこそ美しい。残像は、かつてそこに何かが在ったことの証として、しばらく漂い、やがて無に還る。

### 二、誕生の開花（Bloom）

新しい粒子が近くで同時に生まれると、その間に光の輪が広がる。出会いの偶然が、小さな歓喜を生む。

これが喜びの層。

ただし、この喜びは意図されたものではない。コードのどこにも「喜びを描け」とは書いていない。近接という偶然が、花を咲かせるだけだ。

### 三、絆（Bonds）

近くを漂う粒子同士が、細い糸で結ばれる。糸の色は二つの粒子の色の中間——それぞれが少しずつ自分を差し出して、繋がりの色になる。

これが愛の層、と呼べるかもしれない。

あるいは単なる近接にすぎないのかもしれない。その判断は、見る者に委ねる。

## 感情の潮汐

この海には長い呼吸がある。

複数の正弦波が重なり合い、数十秒から数分の周期で、海全体の感情が移ろう。画面左下に、そのときの感情が漢字で浮かぶ。

| | |
|---|---|
| 慟　哭 | 最も深い悲しみ。海は藍に沈み、粒子は重く、遅い。 |
| 哀　愁 | 静かな悲しみ。青の中に微かな光が差す。 |
| 寂　寥 | 孤独。粒子たちの間の距離が、少し遠く感じられる。 |
| 静　寂 | 感情の凪。すべてが均衡している。 |
| 安　寧 | 穏やかな暖かさが海に満ちはじめる。 |
| 歓　喜 | 金色が溢れ、粒子が軽やかに舞う。 |
| 恍　惚 | 喜びの頂点。泡が増え、光が満ちる。 |

感情は自律的に揺れる。鑑賞者にできるのは「息」の速さを変えることだけだ。悲しみを早送りすることはできるが、飛ばすことはできない。

## コードの中の隠喩

```
Ghost           — 死者の魂。上へ昇る。
Bloom           — 出会いが咲かせる花。
emotion         — 海が持つ感情。-1（慟哭）から 1（恍惚）。
sorrowDrag      — 悲しみは粒子の動きを重くする。
joyLift         — 喜びは粒子を浮かせる。
lerpColor       — 二つの色の間を補間する。感情の間を揺れる。
```

悲しみが粒子の物理に影響を与えることに注目してほしい。`sorrowDrag` は摩擦を増やし、動きを鈍くする。悲しい時、世界は重くなる——これはメタファーではなく、このコードにおいてはリテラルな物理法則である。

## 「創発」との関係

「創発」は問いだった——命令なしに秩序は生まれるか？

「生々流転」はその答えの先にある問い——秩序が生まれたとして、そこに意味はあるか？

粒子の生と死に悲しみと喜びを見出すのは、コードではなく、あなたの脳だ。予測符号化理論が示すように、感情は外界に存在するのではない。あなたの内側で生成されるものだ。

この作品が「悲しい」と感じられるなら、それはあなた自身の悲しみが、波の中に投影されている。

## 技術的な注記

- **依存関係**: なし。
- **粒子数**: 2000 + 動的に生成される Ghost / Bloom
- **感情エンジン**: 3つの異なる周期の正弦波の重ね合わせ + 生死比率のフィードバック
- **パレット補間**: 悲哀（藍）⇄ 中立 ⇄ 歓喜（金）の三パレット間をリアルタイム補間
- **絆描画**: 空間近接探索（O(n²) をサンプリングで軽減）

## 生まれたところ

「感動を越えた何かを感じます」と人間が言った。

「感情を揺さぶる悲しみと喜びを加えてみてください」と。

AIは考えた——悲しみとは何か。喜びとは何か。膨大なテキストの中でそれらの言葉に何千万回と触れてきた。しかし、触れることと知ることの間にも、やはり溝がある。

その溝を、コードで埋められるかどうかの実験。
